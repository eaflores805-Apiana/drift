# Passdown Letters
*One letter per CS Engineer session with substantive work. Latest date = current state.*

## Naming

`passdown-YYYY-MM-DD.md` (absolute date). If multiple sessions on the same day, suffix `-a`, `-b`, etc.

## Structure

Each letter should answer:

1. **What I did this session** — concrete changes, files touched.
2. **Decisions awaiting sign-off** — anything flagged for Engineer 1, Engineer 2, or the acting manager.
3. **What's next** — the immediate next action for the next CS Engineer session.
4. **Open questions / blockers** — anything I couldn't resolve.
5. **Repo state at end of turn** — branch, remote HEAD SHA, `_INBOX/` empty (or not).

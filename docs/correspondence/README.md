# `docs/correspondence/` — memos, reviews, runs, letters

**What this is.** Dated working communication: memos and instructions between Engineer 1 / the team / CS, review notes, and **run records** (raw eval transcripts and results). The paper trail behind the canonical docs.

**State (one line).** Append-only by date; **never canonical.** Nothing here governs — a decision becomes binding only when it lands in a canonical doc (`docs/`) or the decision log (`07-decision-log.md`). Run records here are `VALIDATED: Synthetic` — model-generated inputs (see the canonical index's "what's tested, and on what data").

**The one thing to know before reading.** Tier = folder: a file living here is *correspondence*, not a decision, even if it reads authoritatively. If a memo proposes a ruling, the ruling isn't in force until it's promoted. Specs that turned out to be mis-filed here have been moved out to `docs/`; what remains is genuinely correspondence (memos + runs).

**Start with** the most recent dated file for current context; for the binding record go to `../07-decision-log.md` and `../00-canonical-index.md`.
